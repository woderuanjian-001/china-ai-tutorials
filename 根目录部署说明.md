# ⚠️ 重要：robots.txt 和 404.html 必须放在 GitHub Pages 根仓库

## 你的当前架构问题

你的站点 URL 是：
```
https://woderuanjian-001.github.io/china-ai-tutorials/
```

说明你有两种可能的仓库结构之一：

### 情况 A（最可能）：项目仓库 `china-ai-tutorials`
- 仓库：`https://github.com/woderuanjian-001/china-ai-tutorials`
- Pages 部署分支：`main` → `/`
- 这种情况下 `china-ai-tutorials` 仓库根目录的 `public/robots.txt` 实际上对应
  `https://woderuanjian-001.github.io/china-ai-tutorials/robots.txt`（子路径）

### 情况 B：用户主页仓库 `woderuanjian-001.github.io`
- 仓库：`https://github.com/woderuanjian-001/woderuanjian-001.github.io`
- 这里部署的内容才对应 `https://woderuanjian-001.github.io/`（根路径）

## 如何让 `https://woderuanjian-001.github.io/robots.txt`（根路径）返回 200

**Googlebot 默认只抓根路径的 `/robots.txt`**。你的项目仓库的 robots 只会出现在
子路径 `/china-ai-tutorials/robots.txt`，根路径是 404（已实测）。

### 解决方案 1（推荐，一劳永逸）：买独立域名

注册 `china-ai-tutorials.com`（约 ¥80/年），在 GitHub Pages 设置里绑定。
此时根路径就归你了，robots.txt 自然生效。详见附件「联盟营销与变现方案」。

### 解决方案 2（零成本，临时方案）：创建/使用用户主页仓库

1. 登录 GitHub，创建仓库 `woderuanjian-001.github.io`（如果已存在就用现有的）
2. 在该仓库根目录（不是 china-ai-tutorials 仓库！）放入下面两个文件
3. 启用该仓库的 GitHub Pages（Settings → Pages → Source: main / root）

这样 `https://woderuanjian-001.github.io/robots.txt` 就会返回正确内容。
